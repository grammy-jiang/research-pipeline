# Rule Pack: Interface Rules

Apply as a review gate (prompt 21).

- [ ] Interfaces are small and deep (narrow surface, substantial behavior).
- [ ] Every contract includes an error contract (not just the happy path).
- [ ] Every contract includes a versioning expectation.
- [ ] Agent and tool I/O are structured wherever possible (testable contracts).
- [ ] Each interface names its owner and its observability fields.
- [ ] Contracts are stable enough to write contract tests against (§19).

**Gate:** unchecked items block §12 and the contract-test row in §19.
