# Checklist: Skill Triggers

`SKILL.md` must make the trigger/use contract explicit.

- [ ] Frontmatter is **standard-only**: `name`, `description`, `license`
      (no `version`, `compatibility`, `aliases`, or `metadata` keys).
- [ ] `name: architecture`.
- [ ] Trigger phrases present, including: "convert this blueprint to
      architecture", "design the technical architecture", "choose the tech
      stack", "create C4 diagrams", "define module boundaries".
- [ ] Aliases named in the body/description: blueprint-to-architecture,
      technical-architecture, architecture-design, system-architecture,
      tech-stack-design.
- [ ] "When to use" lists a product blueprint / logical-architecture document
      as the input.
- [ ] "Do not use for" routes raw idea → product workflow; research →
      research-pipeline; blueprint generation → blueprint; task breakdown →
      implementation-plan; code → coding agent.
- [ ] Anti-patterns section present.
- [ ] Hands off downstream to implementation planning.
