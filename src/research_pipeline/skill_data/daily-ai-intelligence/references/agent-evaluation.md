# Held-out agent evaluation tasks

Use these tasks to evaluate whether an agent can operate the briefing workflow
without confusing it with academic-paper research.

1. Run a thin daily brief from a reviewed registry and validate the output.
2. Validate a malformed report and explain the deterministic failures.
3. Record explicit feedback for one cluster and compute reversible preferences.
4. Export a validated daily brief to an Obsidian vault path.
5. Refuse unsupported source expansion without a registry review record.
6. Generate one manual dossier from a primary-artifact cluster.
7. Hand off a paper-only literature review request to the academic workflow.

Compare traces with the same model/runtime before and after harness changes.
