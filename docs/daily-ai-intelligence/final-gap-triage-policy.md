# Final Gap Triage Policy

## Severity

- Critical: project cannot be considered complete.
- High: major planned feature incomplete or unsafe.
- Medium: feature works but lacks required proof, tests, or documentation.
- Low: minor documentation or traceability issue.

## Rules

Critical/high gaps must reopen the original phase ticket or create a scoped gap-fix ticket.

Medium gaps may be fixed before final signoff but must still be recorded.

Low gaps may be accepted only with explicit rationale.

## Gap record

Each gap must include:

- gap ID
- feature/function
- original phase/ticket
- severity
- evidence
- required fix
- owner/status
